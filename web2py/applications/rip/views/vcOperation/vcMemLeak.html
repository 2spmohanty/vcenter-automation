VC Memory Leaks

URI: http://{{=response.app_uri}}/rip/vc_0/vcmemleak
This API can be used to identify memory leaks in different services running across the vCenter. The crux of this API is CHAP and AH64, developed by TIM BODDY, VMWare. More details on CHAP would be available in documentation page, where Tim discusses the advantages of using it for Memory analysis. A livecore dump is taken on the service and anlaysis is done using different debuggers to gather the output. This API runs with a disclaimer and caution as coredump might affect the memory/process for quite some time and it can disrupt some critical programs running on the VC. So run it when it is absolutely required to go for memory analysis. Services implemented for analysis are "analytics", "applmgmt","hvc", "imagebuilder", "lookupsvc", "mbcs", "netdumper", "perfcharts", "pschealth","rbd", "rhttpproxy", "sca", "statsmonitor", "trustmanagement","updatemgr", "vcha", "vmcam", "vmonapi", "vmware-postgres-archiver",vmware-vpostgres","vsan-dps","vsan-health", "vsm" . Incase the user needs to run the analysis on all the above service he can specify the "all" value under service key. "vpxd" services Anlysis takes a long time. So there is a separate URI for the VPXD service Leak analysis. Please refer sample JSON.
