VC Memory Growth

URI: http://{{=response.app_uri}}/rip/vc_0/vcMemGrowth
This API can be used to analyse Memory growth in a vcenter. The crux of this API is CHAP and AH64, developed by TIM BODDY, VMWare. More details on CHAP would be available in documentation page, where Tim discusses the advantages of using it for Memory analysis. The Memory growth analysis is done by comparing the counts of Managed Object References in a VC and the instances of MORs in the core file. The corefile is simultaneously taken while the Inventory counts is being done by using VMODL query. The "vc_version" and vc_build" keys are compulsory as it is used to download debug symbols from buildweb. Please refer sample JSON.
